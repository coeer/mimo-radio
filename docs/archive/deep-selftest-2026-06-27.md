## 发现的问题

### Issue #1：DJ TTS 开始前歌曲短暂播放 2 秒
- **轮次**：Round 1
- **现象**：创建会话后，歌曲在 DJ TTS 开场白开始前短暂播放了约 2 秒（时间从 0:00 跳到 0:02），然后 TTS 才开始播放并暂停歌曲
- **根因**：浏览器音频解锁流程中 `setIsPlaying(true)` 在 TTS `setSpeaking(true)` 之前执行，导致歌曲短暂播放
- **影响**：🟡 中 — 用户体验不流畅，开场白前听到 2 秒歌曲片段
- **复现**：创建会话 → 观察歌曲时间 → 在 DJ Speaking 之前会看到 0:01-0:02
- **建议**：在 `page.tsx` 的 `unlockAudio` 中，先检查是否有 introScript，有则先 setSpeaking(true) 再播放 TTS

### Issue #2：快速换歌导致 TTS 串词堆积
- **轮次**：Round 1
- **现象**：快速连续点击"下一首" 5 次，DJ TTS 累积到 2:25（多首串词排队播放），用户需要等 2 分多钟才能听到歌曲
- **根因**：每次换歌都生成 transition TTS，但 `isSpeaking` 守卫阻止新 TTS 打断旧 TTS，导致排队
- **影响**：🟡 中 — 快速切歌后用户长时间听不到音乐
- **复现**：快速点击"下一首" 5 次 → 等 DJ 说完 → 观察 TTS 时长
- **建议**：换歌时取消当前 TTS（调 `stopTTS()`），只保留最新一首的 transition

### Issue #3：XSS 输入未被 sanitize
- **轮次**：Round 2
- **现象**：输入 `<script>alert(1)</script>` 发送后，文本原样显示在聊天区
- **根因**：`sanitizePromptInput` 只处理了 prompt 注入模式（`[system]`、`ignore previous` 等），未清理 HTML 尖括号
- **影响**：🟢 低 — React 的 textContent 不执行脚本，但 promptGuard 应该清理输入
- **复现**：输入 `<script>alert(1)</script>` → 发送 → 检查聊天区
- **建议**：在 `sanitizePromptInput` 中增加 HTML 标签清理（`<` → `&lt;`）

### Issue #4：快速全屏切换 x5 关闭按钮丢失
- **轮次**：Round 4
- **现象**：快速连续打开/关闭全屏播放器 5 次，只记录到 open 动作，close 按钮未找到
- **根因**：全屏播放器的打开/关闭动画可能有延迟，快速切换时 DOM 状态不一致
- **影响**：🟢 低 — 页面保持稳定（PLAYING），用户可再次点击全屏恢复
- **复现**：快速连续点击全屏按钮 5 次
- **建议**：在全屏切换动画期间禁用按钮点击（debounce）

### Issue #5：全屏播放器歌词区无内容
- **轮次**：Round 4
- **现象**：全屏播放器中歌词区未显示歌词（`hasLyric: false`），也没有"歌词加载中"或"还没有 DJ 解说"的提示
- **根因**：网易云歌曲的 LRC 歌词 API 未返回数据，或 `useLyric` hook 未正确加载
- **影响**：🟡 中 — 全屏播放器的歌词区空白，用户体验不佳
- **复现**：打开全屏播放器 → 观察歌词区
- **建议**：检查 `/api/v1/lyric` 端点是否正常工作；确保 `useLyric` 在无歌词时显示兜底文案

### Issue #6：快速收藏/取消触发 429 限流
- **轮次**：Round 1
- **现象**：快速连续点击收藏/取消 10 次，6 个 feedback 请求返回 429（限流）
- **根因**：`generalLimiter` 配置为 200 次/15 分钟，但 feedback 端点没有独立限流，共享全局配额
- **影响**：🟡 中 — 快速操作时部分 feedback 丢失，品味数据不完整
- **复现**：快速点击收藏按钮 10 次 → 检查后端日志 → 看到 429
- **建议**：为 feedback 端点添加独立的 rate limiter（如 30 次/分钟），或在前端做 debounce

### Issue #7：navigate 导致会话完全丢失
- **轮次**：Round 3
- **现象**：从首页导航到 /plan 再回来，会话丢失，回到引导态
- **根因**：sessionId/sessionToken/queue/currentSong 不持久化（设计决策）
- **影响**：🟡 中 — 用户在 /plan 点歌后回到首页需要重新创建会话
- **复现**：创建会话 → 导航到 /plan → 导航回首页
- **建议**：这是已知设计决策。可优化：/plan 页面的播放按钮直接开始播放而不跳转回首页
