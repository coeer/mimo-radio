## 发现的新问题

### Issue A：首次访问 /plan 报 "生成失败"
- **现象**：首次打开 /plan 页面显示"今日电台计划生成失败，稍后重试"
- **原因**：可能是 schedule API 在后端刚启动时超时或返回异常
- **影响**：🟡 中 — 用户首次访问 /plan 看到错误，刷新后正常
- **复现**：直接在新标签页打开 /plan

### Issue B：mini player 因 session 丢失未显示
- **现象**：从首页导航到 /plan 后，mini player 不显示（currentSong 为 null）
- **原因**：sessionId 不持久化，navigate 后 store 状态重置
- **影响**：🟡 中 — P7 的 mini player 设计依赖 currentSong，但 navigate 会丢失
- **建议**：这是已知设计决策。mini player 只在"同会话内"的 /plan 页生效（如 SPA 内部跳转），不适用于全页 navigate

### Issue C：浏览器自动播放拦截导致 TTS 不播放
- **现象**：TTS 合成成功但浏览器不播放（audio 元素被 autoplay 策略阻止）
- **原因**：webbridge 新开的标签页没有用户手势，浏览器阻止所有音频
- **影响**：⚪ 测试环境限制 — 真实用户点击页面后音频正常解锁
- **说明**：这不是代码 bug，是浏览器安全策略。代码中有 unlockAudio 逻辑处理此场景
