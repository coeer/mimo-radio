## 二、还需要完善的地方（按优先级）

### 🟡 中优先级（上线前应做）

#### 1. P2 安全与质量审查（4 项）
| 项 | 现状 | 说明 |
|----|------|------|
| helmet CSP | 未配置 | 后端 index.ts 有 helmet() 但没配 CSP 策略。上线前必须配 |
| WCAG 颜色对比度 | 未检查 | 深色主题部分文字对比度可能不达标 |
| next/dynamic 代码分割 | 未做 | 首屏加载所有组件，弱网下慢。/plan、/profile、/settings 应用 dynamic import |
| 独立 ErrorBoundary | 未做 | 某组件崩溃会白屏。每个路由应有独立 ErrorBoundary |

**判定**：这些是"上线前必做但现在不急"的。项目还在迭代核心体验，提前做会被后续改动打破。**建议上线前批量处理。**

#### 2. F4 isPlaying 架构债
- **现状**：24+ 处直接调 setIsPlaying，缺仲裁层。当前靠 React 批处理 + Zustand 异步更新兜底。
- **风险**：TTS resume vs MediaSession 切歌的竞态（理论存在，暂无可复现 bug）。
- **判定**：**暂缓**（已在 HANDOVER 落档）。触发条件是出现可复现竞态 bug 时再做架构改造。

#### 3. `useAudioPlayer.sideffects.test.ts` 的 5 个 tsc 错误
- **现状**：Song 缺 emotionTags/sceneTags，5 个既有错误。
- **影响**：运行时测试通过，只是类型检查有警告。
- **判定**：低优先级，顺手修（给 Song mock 补 emotionTags/sceneTags 字段即可）。

### 🟢 低优先级（锦上添花或需外部环境）

#### 4. QQ 音源完整链路验证
- 需 webbridge 打开 y.qq.com 登录 tab 后端到端测试。
- 当前自测全程网易云，QQ 集成（封面/播放 URL/歌词）未实测。

#### 5. ASR 语音输入端到端
- 后端 schema 已修（支持 webm），前端录音链路已接入。
- 但 webbridge 无法模拟麦克风，真实验证需浏览器手动测。

#### 6. MediaSession 锁屏/耳机控制
- 代码已实现，但需真实移动设备验证（webbridge 无法模拟锁屏）。

#### 7. UPnP 音响推送 / 歌单导入
- 功能代码存在，依赖外部硬件/数据，未端到端测试。

#### 8. 净分品味排序（like/skip 冲突）
- 同一歌手可能既被 like 又被 skip（用户喜欢周杰伦但跳过某首）。
- 当前 getLikedArtists/getDislikedArtists 分别统计，DJ 可能困惑。
- **低成本优化**：按净分（like - skip）排序，净分为负的从 liked 剔除。

### ❌ 已否决（grill-me 审议，不建议做）
| 项 | 否决理由 |
|----|---------|
| SSE 流式文本 | 解决的不是真实瓶颈（AI 生成速度才是），成本高收益模糊 |
| WebSocket 实时推送 | LRC 已按时间戳本地高亮，不需要服务器推送；瓶颈是 AI 不是传输 |
| 上下文推荐（天气/时间融入搜索） | 用户感知弱，DJ 串词已承担上下文感 |
| 波形 RMS（替代 sin 降级） | CORS 可能根本解不了，收益低，当前降级（sin+noise+kick）可接受 |
