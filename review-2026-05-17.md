### 2. `catch {}` 吞掉错误

**位置**:
- `frontend/src/app/page.tsx:154`
- `frontend/src/app/page.tsx:201`
- `frontend/src/store/radioStore.ts:133`

**问题**: 空 `catch {}` 不记录任何错误信息，开发时 API 报错完全看不到。

**修复**:
```typescript
// 修复前
catch {
  set({ error: '操作失败' });
}

// 修复后
catch (err) {
  console.error('[Session] API 调用失败:', err);
  set({ error: '操作失败' });
}
```

### 3. `nextSong` 竞态条件

**位置**: `frontend/src/store/radioStore.ts:100-128`

**问题**: `nextSong` 在 API 调用前通过闭包捕获 `queue`，调用完成后用这个可能过期的 `queue` 做去重判断。如果在 API 调用期间 queue 被其他操作修改，会导致重复添加或遗漏。

**修复**:
```typescript
// 修复前（闭包捕获）
const { queue } = get();
// ... API 调用 ...
set({ queue: [...queue, newSong] });

// 修复后（每次读最新状态）
// ... API 调用 ...
const { queue } = get();  // 在 set 之前重新读取
if (!queue.find(s => s.id === newSong.id)) {
  set({ queue: [...queue, newSong] });
}
```

### 4. 请求超时中间件注册顺序错误

**位置**: `backend/src/index.ts:107-114`

**问题**: 超时中间件在路由之后注册。Express 中间件按注册顺序执行，路由已经处理完请求后，超时中间件永远不会触发。

**修复**: 将超时中间件移到 `app.use('/api', ...)` 之前：
```typescript
// 在路由注册之前
app.use((req, res, next) => {
  req.setTimeout(30000, () => { ... });
  next();
});

// 然后注册路由
app.use('/api', radioRoutes);
```

### 5. `loadMockSongs()` 模块导入时执行

**位置**: `backend/src/routes/radio.ts:21`

**问题**: `loadMockSongs()` 在模块顶层调用，每次 import 该模块都会执行。测试时会产生副作用，且在 DB 初始化之前就运行。

**修复**: 改为惰性初始化，在首次请求时加载：
```typescript
let songsLoaded = false;

router.post('/create', ..., async (req, res) => {
  if (!songsLoaded) {
    loadMockSongs();
    songsLoaded = true;
  }
  // ...
});
```

## 三、中优先级问题（🟡 应优化）

### 6. `currentTime` 高频更新导致全组件重渲染

**位置**: `frontend/src/app/page.tsx`

**问题**: `currentTime` 每秒更新约 4 次（`timeupdate` 事件），整个 550 行组件每次都重渲染。

**建议**:
- 提取 `PlayerBar` 组件并用 `React.memo` 包裹
- `currentTime` 仅在 `PlayerBar` 内部订阅
- 其他子组件（ChatArea、QueueList 等）不受影响

---

### 7. Prompt 清理不一致

**位置**: `backend/src/services/mimo.ts:89, 119-127, 137, 152`

**问题**: 部分地方用 inline `.replace(/[<>]/g, '')` 做清理，而非项目已有的 `sanitizePromptInput()`（`utils/promptGuard.ts`）。两套逻辑不一致，inline 版本防护不完整。

**修复**: 统一使用 `promptGuard.ts` 中的 `sanitizePromptInput()`。

---

### 8. `KimiCard` 硬编码颜色

**位置**: `frontend/src/components/KimiCard.tsx:145-154`

**问题**: `song-info-card` 内使用硬编码的 `#1a1a1a`、`#555555`、`#888888`，不跟随主题切换。

**修复**: 改用 CSS 变量：
```css
.song-info-card {
  color: var(--text-primary);
}
.song-info-card .subtitle {
  color: var(--text-secondary);
}
```

---

### 9. `db/index.ts` 使用 `console.log`

**位置**: `backend/src/db/index.ts:56, 152, 169`

**问题**: 项目已有结构化 `logger` 工具（`utils/logger.ts`），但 DB 层直接用 `console.log`。

**修复**: 替换为 `import { logger } from '../utils/logger'`。

---

### 10. 无 `React.memo`，子组件跟随父组件重渲染

**位置**: `frontend/src/components/*`

**问题**: `OnAirBadge`、`DotMatrixClock`、`QueueList`、`TerminalLog` 等纯展示组件均未用 `React.memo` 包裹，父组件任何 state 变化都会导致它们全部重渲染。

**建议**: 对不依赖频繁变化状态的组件添加 `React.memo`。
